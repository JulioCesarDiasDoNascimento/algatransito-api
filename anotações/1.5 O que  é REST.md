# Anotações 
## O que é REST
> REST = Representational State Transfer (Transferência de Estado Representacional)

* Surgiu nos anos 2000 (Roy Fielding)
* É uma api

> Por que usar REST: 
> * Separação entre cliente servidor 
> * Escalabilidade
> * Independência de linguagem
> * Mercado


>  Constraints
> * Cliente-servidor
> * Stateless (A aplicação não tem estado)
> * Cache (Mantem dados)
> * Interface uniformes
> * Sistemas em camadas
> * Código sob demanda
