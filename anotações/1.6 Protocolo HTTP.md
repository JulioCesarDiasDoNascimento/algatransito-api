# Anotações 
## Protocolo HTTP

> Requisição <-> Resposta


> [MÉTODO] [URL] [HTTP/VERSÃO DO PROTOCOLO] [CABECALHOS] [CORPO DA REQUISIÇÃO]


> Ex: POST /produtos HTTP/1.1 content-type: application/json accept: application/json
> { 
> "nome": "Notebook i7,"
> "preco": 2100.0
> }

### Métodos HTTP
* GET - Buscar
* POST - Criar
* PUT - Editar ou Atualizar
* DELETE - Deletar


URI é o caminho!

#### JSON
* Formato leve de troca de dados
* Fácil de ler e escrever
* Estrutura de chave-valor
* Suporta tipos de dados básicos (string, número, booleano, array, objeto)


### Resposta(pode ter ou não corpo)
> [HTTP/VERSÃO DO PROTOCOLO] [CÓDIGO DE STATUS] [MENSAGEM DE STATUS] [CABECALHOS] [CORPO DA RESPOSTA]

> Ex: HTTP/1.1 201 Location: /produtos/331 content-type: application/json
> { 
> "codigo": 331,
> "nome": "Notebook i7,"
> "preco": 2100.0
> }
